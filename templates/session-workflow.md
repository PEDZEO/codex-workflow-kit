# Session Workflow

## 1. Start

- Прочитать `CODEBASE_INDEX.md`.
- Прочитать `.codex/memory/current-task.md`.
- При необходимости прочитать `.codex/memory/decisions.md`.

## 2. Search

- Найти кандидатов через `rg --files`.
- Уточнить через `rg`.
- Читать только нужные файлы и только нужные диапазоны.

## 3. Plan

- Сформулировать цель в 2-5 строках.
- Выделить критический путь.
- Выделить задачи для параллельных агентов.

## 4. Execute

- Критический путь выполнять локально.
- Сайд-задачи отдавать агентам с явной зоной ответственности.

## 5. Validate

- Проверить diff.
- Запустить минимальную релевантную проверку.
- Убедиться, что задача действительно закрыта.

## 6. Save State

- Обновить `current-task.md`.
- Перенести завершенное в `done.md`.
- Если решение влияет на будущие задачи, записать его в `decisions.md`.
