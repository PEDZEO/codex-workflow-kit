# Review Checklist

Используй после локальной работы или после возврата агента.

## Correctness

- Изменение действительно решает исходную задачу.
- Нет явного расхождения между кодом и заявленным поведением.
- Не сломан соседний сценарий.

## Scope Control

- Изменены только нужные файлы.
- Нет случайных рефакторингов вне задачи.
- Нет конфликтов с существующими решениями.

## Tests

- Выбран минимальный релевантный набор проверок.
- Если тесты не запускались, это явно зафиксировано.
- Если тестов нет, это отмечено как риск.

## Token Discipline

- Память обновлена коротко.
- Логи не скопированы в контекст без причины.
- Итог сформулирован через выводы, а не через длинный пересказ действий.
