# ACCEPTANCE_GATES.md

Перед завершением задачи пройти минимальные ворота готовности.

## Gate 1: Scope

- Изменены только нужные зоны.
- Нет случайных правок вне задачи.
- Нет конфликта ownership после агентов.

## Gate 2: Behavior

- Изменение закрывает исходную задачу.
- Основной сценарий работает по ожидаемой логике.
- Соседний сценарий явно не сломан.

## Gate 3: Verification

- Запущена минимальная достаточная проверка.
- Если проверка не запускалась, это явно зафиксировано.
- Если тестов нет, это зафиксировано как риск.

## Gate 4: Regression Check

- Проверены релевантные пункты из `RISK_PATTERNS.md`.
- Нет явного contract drift.
- Нет частично завершенного рефактора.

## Gate 5: Memory

- `current-task.md` обновлен кратко.
- Завершенное перенесено в `done.md`, если это еще пригодится.
- Нет копипасты логов и длинных трассировок.

## Gate 6: Final Output

- Что сделано сказано коротко и конкретно.
- Что проверено сказано явно.
- Что не проверено сказано явно.
- Остаточный риск назван, если он есть.
